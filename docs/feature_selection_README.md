# 特征筛选说明（Feature Selection）
## 1. 特征筛选目的

由于原始特征数量较多，且部分特征可能存在：

- 缺失值过高
- 取值变化极小
- 特征之间高度相关
- 对模型贡献较低

因此需要对特征进行筛选。

本项目所有特征筛选操作仅基于训练集完成，以避免导致数据泄露的问题。

## 2. 特征筛选流程

本项目采用以下三步进行特征筛选：

- Step1: 去除无效特征
- Step2: 消除冗余
- Step3: 排序

## 3. Step1：去除无效特征

### 3.1 缺失率过滤

首先统计各特征缺失率，对于缺失率超过 50% 的特征，认为：

- 信息不完整；
- 对模型贡献有限

因此进行删除。

缺失率计算公式：

```python
missing_ratio = X.isnull().mean()
```

保留缺失率低于阈值的特征：

```python
missing_ratio < 0.5
```

### 3.2 低方差过滤

对于数值型特征，进一步进行方差过滤。

若某特征取值几乎不发生变化，则：

- 无法有效区分样本
- 对模型贡献较低

因此删除低方差特征。

本项目使用：

```python
VarianceThreshold(threshold=0.01)
```

进行过滤。

仅对数值型特征执行该步骤：

```python
numeric_cols = X.select_dtypes(include=[np.number]).columns
```

过滤后保留有效特征用于后续建模。


## 4. Step2：消除冗余

### 4.1 过滤目的

部分特征之间可能存在高度相关性。

例如：

- total_views 与 last_7day_pv
- total_buy 与 buy_rate

若多个特征表达的信息高度重复，则：

- 会增加特征冗余
- 降低模型可解释性

因此需要进行相关性过滤。

### 4.2 相关系数计算

本项目使用：

```python
corr_matrix = X[numeric_cols].corr().abs()
```
进行计算，仅对数值型特征进行相关性分析。

### 4.3 删除规则

当两个特征相关系数：>= 0.8 时，认为其存在较强相关性。

默认保留：

- 业务价值更高
- 可解释性更高
- 更稳定

的特征，同时删除冗余特征。

### 4.4 关于模型和校验

本项目以树模型作为主模型。

由于树模型对多重共线性不敏感，因此：

- 不再额外进行 VIF 校验；
- 仅通过相关系数过滤控制特征冗余。

## 5. Step3：排序

### 5.1 排序目的

经过前两步后，仍可能存在：

- 对模型贡献较低
- 信息量不足

的特征。

因此进一步使用LightGBM进行特征重要性排序。

## 5.2 方法

使用轻量级LightGBM模型进行训练：

```python
importance_type='gain'
```

使用 Gain 作为特征重要度指标，Gain 值越高，特征对模型预测越重要。

### 5.3 特征保留策略

按照特征重要性从高到低排序。

保留累计贡献达到：95% 以上的特征，同时删除其余低贡献特征。

### 5.4 排列重要性校验

对于核心特征，进一步在验证集上使用排列重要性进行二次验证。

用于检查：

- 特征是否真正提升模型效果
- 是否存在伪重要特征

## 6. 数据泄露控制

为了避免未来信息泄露：

- 所有特征筛选
- 相关性计算
- 特征重要性排序

均仅基于训练集完成，验证集与测试集仅用于模型评估。

## 7. 特征筛选结果

经过特征筛选后：

- 删除低质量特征
- 降低特征冗余
- 保留核心高价值特征

最终形成用于模型训练的特征集合。
