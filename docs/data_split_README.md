# 数据集划分记录（Dataset Split）
## 一、划分目的
为了避免数据泄露，本项目采用基于时间窗口的方式划分训练集、验证集与测试集。
相比随机划分，时间划分更符合真实业务场景中的用户行为预测流程。

## 二、时间窗口划分
### 具体划分如下：

| 数据集 | 时间范围 |
|---|---|
| Train | 2014-11-18 00:00:00 ~ 2014-12-10 19:00:00 |
| Validation | 2014-12-10 19:00:00 ~ 2014-12-15 20:00:00 |
| Test | 2014-12-15 20:00:00 ~ 2014-12-18 23:00:00 |


## 三、Label分布验证
### 对 Train / Validation / Test 中正样本（label=1）比例进行统计：

| 数据集 | 正样本比例 |
|---|---|
| Train | 0.01603 |
| Validation | 0.02020 |
| Test | 0.01596 |

### 结果表明：
- 三个数据集的 label 分布较为接近
- 不存在明显的data distribution shift
- 当前时间窗口划分具有合理性

## 四、时间分布验证
检查训练集中的时间覆盖情况。
### 每日样本分布
训练集覆盖多个日期，同时包含工作日与周末数据，能够较好反映真实用户行为模式。
### 每小时样本分布
训练集在24小时内均包含样本，能够较好反映真实用户行为模式小时。

其中：
- 晚间（20:00 ~ 23:00）用户行为最活跃；
- 凌晨时段样本量较少，但仍保持覆盖。

因此，当前划分能够较好保证不同时间段行为特征的完整性。

## 五、数据泄露控制
- 所有特征筛选、统计分析与模型训练仅基于训练集完成。
- 验证集与测试集仅用于模型评估，避免数据泄露。
